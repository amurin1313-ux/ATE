# Структура приложения ATE v3.2

Ниже — актуальная структура репозитория с назначением каждого ключевого блока.

## 1) Корень проекта
- `.gitignore` — правила исключения служебных файлов из Git.
- `README.md` — краткое описание, запуск, базовые инструкции.
- `INTEGRATION_AND_TEST.md` — чек-лист интеграционных проверок.
- `requirements.txt` — Python-зависимости.
- `ATE_Desktop.spec` — конфигурация PyInstaller для сборки EXE.
- `build_exe.bat` — основной сценарий сборки Windows EXE (создание venv, установка зависимостей, сборка).
- `Структура.md` — этот документ.
- `install_deps.bat` — ручная установка зависимостей в venv (для dev-цикла).
- `run_app.bat` — запуск UI из исходников.
- `run_tests.bat` — запуск тестов из исходников.

## 2) Приложение (UI)
- `app/__init__.py` — пакет UI.
- `app/main.py` — точка входа desktop-приложения (Tkinter), загрузка/инициализация конфига, запуск EngineController.

## 3) Ядро торговой логики
Папка `engine/`:
- `controller.py` — главный оркестратор торгового цикла.
- `controller_lifecycle.py`, `controller_runtime.py` — разнесённые части жизненного цикла и runtime-процессов контроллера.
- `symbol_channel.py` — изолированный контур обработки символа.
- `strategy_runtime.py` — создание/вызов стратегии по символам.
- `trader.py`, `portfolio.py` — операции исполнения и состояние позиций/портфеля.
- `market_regime.py` — рыночные фильтры/режимы.
- `symbol_universe.py` — управление набором торгуемых символов.
- `banlist.py` — баны символов и счётчики ошибок.
- `decision_logger.py`, `snapshots.py`, `metrics.py`, `logging_utils.py` — наблюдаемость, логи, метрики, снапшоты.
- `config.py` — загрузка/сохранение и нормализация конфига.
- `cache.py`, `safe_exec.py`, `migration.py`, `auto_trader.py` — вспомогательные механизмы ядра.
- `version.py` — единый источник правды по версии приложения.

## 4) Интеграция с OKX
Папка `okx/`:
- `public_client.py`, `private_client.py` — REST-клиенты.
- `ws_public.py`, `ws_private.py` — WebSocket-клиенты.

## 5) Стратегии
Папка `strategies/`:
- `strategy_v3.py` — текущая рабочая стратегия.

## 6) Данные и runtime-конфиги
Папка `data/`:
- `config.json` — основной runtime-конфиг приложения.
- `per_symbol_thresholds.json` — пороги по символам.
- `okx_symbol_universe.txt` — список инструментов/универсума символов.

## 7) Инструменты сопровождения
Папка `tools/`:
- `check_version_consistency.py` — проверка согласованности версии по проекту.
- `replay_decisions.py` — офлайн-реплей сигналов по decision logs.
- `analyze_session.py` — анализ результатов сессии.
- `generate_thresholds_from_dist.py` — генерация per-symbol порогов из логов.

## 8) Тесты
Папка `tests/`:
- `test_metrics.py` — проверка расчётов метрик.
- `test_signing.py` — проверка подписи API.
- `test_okx_multifill_manual_path.py` — сценарий multi-fill.

## 9) Документация
Папка `docs/`:
- `STRATEGY_V3_DESCRIPTION_RU.md` — подробное описание StrategyV3.
- `REPO_AUDIT_RU.md` — аудит структуры/legacy и план улучшений.
- `DEEP_ANALYSIS_AND_PLAN_RU.md` — детальный план доработки стратегии/архитектуры.
