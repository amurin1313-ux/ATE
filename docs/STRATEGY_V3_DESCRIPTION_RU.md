# StrategyV3 — детерминированное правило-ядро (Score)

## Ключевая идея
StrategyV3 **не** вычисляет «вероятность» и не использует p_buy/p_sell.
Вместо этого она вычисляет **Score (0..1)** — насколько текущие рыночные условия совпали с набором правил входа/выхода.

## Входные метрики
Стратегия использует метрики из `engine/metrics.py`, которые каждую итерацию попадают в пакет `features`:
- RSI(14)
- MACD histogram
- ATR(14) в процентах
- Spread(%)
- наклон цены (slope) на окне
- доходности ret_5/ret_15
- buy_ratio / дисбаланс
- volume ratio (объём к SMA)

## Пороги
Пороги задаются **по каждой монете отдельно** в `data/config.json` → `trading.per_symbol_thresholds`.
Также есть глобальные пороги исполнения:
- `trading.v3_buy_score_min`
- `trading.v3_sell_score_min`

## Выход
Стратегия возвращает:
- `action`: BUY/SELL/HOLD
- `confidence`: **Score** (оставлено поле name=confidence только ради совместимости интерфейсов)
- `reason`: текст причины
