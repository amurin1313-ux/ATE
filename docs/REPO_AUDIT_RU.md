# Аудит проекта ATE (кратко)

## 1) Структура репозитория
- `app/` — desktop UI (Tkinter), загрузка/валидация конфигурации, запуск контроллера.
- `engine/` — ядро торговой логики: контроллер, жизненный цикл, метрики, кеши, режим рынка, снимки состояния.
- `okx/` — REST/WS клиенты для OKX (public/private).
- `strategies/` — стратегии принятия решений (в текущей версии используется `StrategyV3`).
- `tools/` — утилиты диагностики/реплея/проверки версий и калибровки порогов.
- `tests/` — модульные тесты (метрики, подпись API, сценарий multi-fill).
- `data/` — runtime-конфиги и справочные данные (включая `config.json`).
- `docs/` — документация по стратегии.

## 2) Найденные артефакты старых версий
1. В репозитории находились кэш-артефакты интерпретаторов и тестов:
   - множество `__pycache__/*.pyc`;
   - `pytest_cache/`.
2. В коде сохранились «исторические» комментарии, не соответствующие активной стратегии:
   - в `engine/strategy_runtime.py` упоминается `Strategy V5`, хотя фактически зарегистрирована `StrategyV3`.
3. В `data/config.json` есть дополнительное поле `runtime.version = "3.1.5.0"`, которое выглядит как legacy-след и может путать при диагностике.

## 3) Найденные ошибки в работе
1. **Нарушение консистентности версий в документации**:
   - `tools/check_version_consistency.py` падал из-за несовпадения версии в заголовке `README.md`.
2. Тестовый набор (`pytest`) проходит, критичных падений в покрытых сценариях не обнаружено.

## 4) План исправления (после изучения)
1. **Гигиена репозитория**
   - удалить кэш-артефакты из Git и запретить их повторное попадание через `.gitignore`.
2. **Единый источник версии**
   - использовать `engine/version.py` как единственный источник;
   - валидировать `README.md` и `data/config.json` в CI через `tools/check_version_consistency.py`.
3. **Убрать legacy-шум**
   - привести комментарии к текущей реальности (`StrategyV3`), чтобы сократить риск ошибочной поддержки «мертвого» кода.
4. **Укрепить диагностику**
   - добавить smoke-check перед релизом: `pytest -q`, `python tools/check_version_consistency.py`, `python -m compileall ...`.
5. **Дальнейший техдолг**
   - ревизия дублирующихся/legacy-полей конфигурации (`runtime.version` и близкие) с миграцией и обратной совместимостью.
