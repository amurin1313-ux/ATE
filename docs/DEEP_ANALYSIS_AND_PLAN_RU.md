# Глубокий анализ и план доработки ATE 6PRO (v3.2)

## 0) Что сделано в этой итерации (фактические изменения)
1. Исправлена критичная ошибка в `engine/portfolio.py` при обработке внешней продажи без найденного BUY:
   - вместо несуществующего поля `reason` теперь используется корректное `sell_reason`;
   - убрана ссылка на `tr` внутри ветки `if tr is None` (исключение и «проглатывание» ошибки).
2. Укреплён расчёт итогового и мгновенного PnL:
   - комиссии теперь обрабатываются через `abs(...)`, чтобы корректно учитывать формат OKX (часто `fee` приходит отрицательным);
   - исправления внесены как в `est_pnl_now`, так и в `realized_pnl`.
3. Добавлены тесты:
   - проверка PnL при отрицательных значениях комиссий (`quote`/`base`);
   - проверка сценария orphan SELL (внешняя продажа без открытой сделки).
4. Восстановлены утилитарные BAT-скрипты (`install_deps.bat`, `run_app.bat`, `run_tests.bat`) для стабильного сценария сопровождения и повторяемой ручной валидации.
5. Добавлен инструмент `tools/analyze_market_thresholds.py` для калибровки порогов на реальных рыночных данных (публичные endpoints OKX), с устойчивой обработкой сетевых ограничений.

## 1) Диагностика текущего состояния
### 1.1 Логи и runtime-данные
- В текущем окружении отсутствуют каталоги `data/decision_logs` и `data/logs`, поэтому провести ретроспективный анализ реальных сигналов BUY/SELL за последний сеанс не удалось.
- Из-за этого нельзя верифицировать конкретные кейсы вида «монета выросла, а входа не было» по таймстампам внутри decision log.

### 1.2 Реальный рынок (OKX)
- Подготовлен анализатор `tools/analyze_market_thresholds.py`.
- Попытка запросить live-данные с OKX выполнена, но в окружении прокси-блокировка (`403 Forbidden`), поэтому автоматическая калибровка порогов по live-метрикам в этой сессии ограничена.
- Снапшот с ошибками сохранён в `data/market_thresholds_snapshot.json`.

### 1.3 Основные зоны риска, выявленные по коду
1. Ошибки «тихого» падения при edge-сценариях синхронизации fills (исправлено).
2. Потенциально неверный учёт комиссии из-за знака `fee` (исправлено).
3. Недостаток автодиагностики по рынку/порогам без decision logs (частично закрыто новым инструментом).

## 3) Codex-инструкции (рабочий шаблон)
1. Сначала чинить P0-баги (исполнение и PnL), потом P1 (стратегия), затем P2 (оптимизация).
2. Любая новая метрика в стратегии обязана:
   - попасть в meta,
   - попасть в decision logs,
   - быть отображена в UI.
3. Любое новое условие блокировки — с кодом `BLOCK_*` и читаемым русским текстом.
4. Не смешивать «факт сделки» и «сигнал»: факт подтверждается только после fill.
5. Перед merge всегда запускать:
   - `pytest -q`
   - `python tools/check_version_consistency.py`
   - `python -m compileall -q app engine okx strategies tools tests`
6. Для изменений порогов обязательно прикладывать:
   - краткий отчёт replay/live-анализа,
   - какие пороги изменены,
   - ожидаемый эффект на winrate/частоту входов.
